Un catalogo unico per gestire tutte le scene senza boilerplate.

Le scene restano separate in file JavaScript autonomi, ma nav, gallery e viewer sono guidati da un catalogo centrale per evitare HTML duplicati e routing manuale.

Guida Rapida

Come usare e integrare queste animazioni

Animation Vault e' una raccolta di scene Three.js separate, ciascuna con pagina dedicata, file JavaScript autonomo e asset condivisi. L'obiettivo e' poterle provare qui e spostarle facilmente in un altro progetto.

Come funziona

  • La home raccoglie tutte le animazioni disponibili.
  • Un solo viewer generico apre qualsiasi scena tramite query param.
  • Ogni `src/scenes/animX.js` contiene la logica Three.js della scena.
  • Header e card della home leggono lo stesso catalogo centrale.
  • Texture e modelli vivono in `public/textures/` e `public/models/`.

Avvio locale

  1. Installa le dipendenze con `npm install`.
  2. Avvia il progetto con `npm run dev`.
  3. Crea la build con `npm run build` quando serve.

Integrare una animazione in un altro progetto

  1. Installa `three` nel progetto di destinazione.
  2. Copia il file scena che ti interessa da `src/scenes/animX.js`.
  3. Crea un container HTML con l'id richiesto, ad esempio `animX`.
  4. Copia gli asset necessari da `public/textures/` o `public/models/`.
  5. Aggiorna i percorsi degli asset se cambiano cartelle o URL.
  6. Monta la scena dentro la tua route o componente mantenendo il container corretto.

Cosa controllare dopo la copia

  • Il container della scena deve esistere davvero nel DOM.
  • I file statici devono mantenere percorsi validi.
  • Camera e renderer devono leggere le dimensioni del container giusto.
  • Gli import `three/examples/jsm/...` devono essere supportati dal bundler finale.

Aggiungere una nuova scena qui dentro

  1. Crea `src/scenes/animX.js`.
  2. Aggiungi metadati e descrizione nel catalogo `src/content/animations.js`.
  3. La home, la nav e il viewer la leggono automaticamente.
